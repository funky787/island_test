### Юнит-тестирование

Юнит-тесты проверяют отдельные компоненты системы изолированно.

#### Покрытые классы

**Island**
- корректное создание карты  
- корректная инициализация локаций  
- выброс исключения при выходе за границы  

**Location**
- добавление растения  
- добавление и удаление животного  

**Animal**
- корректная обработка координат (clampX, clampY)  
- увеличение голода при step()  
- смерть при превышении порога голода  
- удаление из локации после смерти  

**Питание**
- травоядное поедает растение  
- хищник поедает травоядное  
- хищник не поедает других хищников  
- отсутствие размножения при отсутствии пары  

---

### Функциональное тестирование

Функциональные тесты проверяют работу системы целиком.

Проверено:

- корректное выполнение шага симуляции (Island.step())  
- вывод статистики после шага  
- корректный вывод карты острова (printMap())  

---

### Использованные техники тест-дизайна

**Граничные значения**
- отрицательные координаты  
- координаты за пределами карты  
- проверка методов clamp  

**Классы эквивалентности**
- валидные координаты  
- невалидные координаты  

**Тестирование переходов состояний**
- жив -> голод увеличивается -> смерть -> удаление из локации  

**Сценарное тестирование**
- полный шаг симуляции  
- проверка вывода в консоль
